2、一个完整的项目的开发流程
	产品人员、UI设计人员：需求分析，产品定位，市场调查，...UI设计师按照产品的规划设计出对应的效果图(PSD -> PHOTOSHOP)
	前端开发工程师
	后端开发工程师
	
3、crm系统功能分析
	(1)获取所有的客户信息
	(2)把页面中新填写的信息获取到，增加新的客户(把新的客户信息存储到服务器上)
	(3)获取某一条指定客户的信息，把信息放入到页面的对应位置
	(4)把原有客户的某些信息进行修改(把服务器上存储的那一份信息删掉)
	(5)删除指定的客户信息(把服务器上之前存储的某一条删除掉)

4、服务器后台开发
	(1)首先需要在服务器上搭建一个服务(监听一个端口)，把我们的项目发布出去。
	(2)根据客户端的请求把所有的客户信息准备好，并且返回给客户端
	(3)把客户端传递进来的新的客户信息增加到服务器的指定位置上(正常情况下都是存储咋数据库当中，我们今天的案例用文件存储代替数据库存储)，并且告诉客户端增加成功还是失败

5、客户端前端开发
	(1)拿到PSD效果图后，先把HTML + CSS 写完
	(2)通过AJAX向服务器端发送一个请求，把数据获取到，解析和数据绑定
	(3)把页面中用户输入的信息获取到，并且通过AJAX传递给服务器，接受服务器端高度我们成功还是失败，从而进行后续的操作
	
6、API接口文档
	客户端和服务器端的内容数据的传输一般都是通过JSON格式的字符串进行的
	(1)获取所有的客户信息
		URL:"/getList";获取信息的地址
		GET：请求的方式
		参数：(客户端传递给服务器的内容) 无
		返回值：(服务器端返回给客户端的内容)	
		{
			'code':0	//返回结果标识的描述：0代表成功，1代表失败(没有任何客户信息)
			"msg":"获取成功"，//返回结果标识的说明信息
			"data":["id":xxx,"name":"xxx","age":xxx]
			
		}
	(2)(1)获取具体某一位客户信息
		URL:"/getInfo";获取信息的地址
		GET：请求的方式
		参数：?id=xx(客户端传递给服务器的内容) 
		返回值：(服务器端返回给客户端的内容)	
		{
			'code':0	//返回结果标识的描述：0代表成功，1代表失败(没有任何客户信息)
			"msg":"获取成功"，//返回结果标识的说明信息
			"data":{
				"id":xxx,
				"name":"xxx",
				"age":xxx,
				"phone":"xxxxxxxxx",
				"address":"xxxxxxx"	
			}
		}
	(3)增加客户信息
		URL:"/addInfo"
		POST
		参数：请求主体中 {
			"namae":"xxx",
			"age":"xxx",
			"phone":"xxx",
			"address":"xxxxxx"
		}
		(少了一个ID，在增加的过程中，我们的ID是服务器端自动按照规律生产的：在现有的最后一个客户ID的基础上+1即可)
		
		返回值：{
			"code":0,//0成功，1失败
			"msg":""
		}
	(4)修改客户信息
		URL:"/updateInfo" 
		POST
		参数：请求主体中 '{"id":xx,
			"name":"xxx",
			"age":xxx,
			"phone":xxxxx,
			"address":"xxxx",
			}'
		返回：{
			"code":0 //0成功，1失败
			"msg":""
		}
	5、删除客户信息
		URL:"/removeInfo" 
		GET 
		参数:？id=xxx;
		返回：{
			"code":0 //0成功，1失败
			"msg":""
		}
