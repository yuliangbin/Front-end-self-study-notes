REM 响应式布局
	我们做的H5页面只是在移动端访问(REM 不兼容低版本的浏览器)
	
第一步：从UI设计师那里拿到设计稿(PSD格式的设计稿) 640*1136

第二步：在样式中给HTML设定一个fontSize的值，我们一般都设置一个方便后面计算的值。例如:10px,100px,这里之所以设置100px，是因为浏览器最小的字体都是100px。用10px比例计算的结果会和真实的UI设计稿存在那么一点点差距
	
第三步：写页面 写样式
	完全按照设计稿的尺寸来写样式，此时我们不需要管任何事情，设计稿给你的宽度、高度、字体大小、margin、padding是多少，我们就写多少。
	但是当我们写样式值的时候，需要把得到的像素值除以100，计算出对应的REM的值，我们设定的值也是REM的值
	(值得注意的是:真实项目中外层的盒子的宽度我们一般不写固定值，沿用流式布局的思想，我们用百分比的布局方式)
	
第四步:根据当前屏幕的宽度和设计稿的宽度来计算我们的HTML的fontSize的值
	设计稿：640		600*300		fontSize=100		6rem*3rem
	手机：	320		300*150		fontSize=50			
	手机：	375		(375/640)*100
	
	所以需要根据屏幕宽度和设计稿的宽度的比例，动态计算一下当前宽度下的fontSize值应该是多少，如果fontSize的值改变了。之前设定的所有rem单位的值自动会跟着放大或缩小
	function () {
		var desW = 640,
			winW = document.documentElement.style.fontSize,
			ratio = winW / desW;
		document.documentElement.style.fontSize = ratio * 100 + "px";
	}
	
	当浏览器窗口大于设计稿的宽度的时候，为了保证页面的整体效果(防止图片、文字的伸缩)，我们让页面等于设计稿的宽度.
